MongoDB 설치

가장 보편적으로 사용하는 NoSQL인 MongoDB의 설치 방법을 알아보자.

‘OpenTSDB’, ‘InfluxDB’ 또한 NoSQL이지만 ‘MongoDB’를 먼저 선택하게 된 계기가 있다. mongoDB의 데이터는 각 ‘collection’에 ‘document’ 형식으로 저장이 되는데, 이는 파이썬 ‘Dictionary’ 자료형과 같다.

데이터 분석시 파이썬 ‘Dictionary’ 자료형을 많이 이용하기 때문에 나의 첫 NoSQL을 몽고DB로 정했다.?
(제조업의 주 데이터인 시계열 데이터를 관리하기에 InfluxDB가 좋다고 하나 이후에 알아보도록 하겠다!)

설치파일을 다운받을 수 있는 URL이다.
[ https://www.mongodb.com/download-center/community?jmp=docs ]


본인에게 맞는 사양을 선택하세요!

다른 설치 프로그램들과 같이 쭈욱 설치를 진행하며 설치 경로 변경이 필요하다면 ‘Custom’ 버튼을 통해 경로를 수정해준다.

CMD 창에서 설치 경로를 찾아 들어가지 않아도 바로 실행할 수 있도록 환경변수를 추가한다.

시스템 변수 중 ‘Path’를 들어가 다음과 같이 bin 경로를 추가한다.


환경변수 추가

이후 몽고DB를 실행시키는 명령인 'mongod'를 입력했을 때 에러가 발생한다.
기본 db 경로인 'C:\data\db\'폴더가 없기 때문이다.

해당 경로로 폴더를 생성해준다. 다른 곳에 생성하고 싶다면 '--dbpath' 명령어를 통해 경로를 바꿔주자.
이후 'mongod' 명령어를 통해 MongoDB를 실행할 수 있다.

웹브라우저에 'localhost:27017' 을 입력하여 다음 메세지가 출력된다면 제대로 작동하는것이다.
'It looks like you are trying to access MongoDB over HTTP on the native driver port.'
